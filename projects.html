In production AdCP

Purrsonality

A cat-personality quiz that doubles as a working AdCP seller agent. Visitors answer a handful of questions, get a result page describing their cat's archetype (Velvet Whisper, Daring Explorer, Tiny Tornado, Sly Trickster, Tiny Tyrant), and along the way the site quietly becomes a real ad publisher.

Under the hood it runs on Astro and Cloudflare Pages with a hand-rolled AdCP request dispatcher, RFC 9421 HTTP message signatures for transport authentication, and signal segmentation derived from quiz answers. It is one of the smallest production AdCP sellers in existence — and a useful proving ground for everything I want to push back upstream.

Public site: purrsonality.rocketscience.pl (alias: purrsonality.pages.dev).

Active Open standard

AdCP — Ad Context Protocol

The Ad Context Protocol is the open spec that lets AI agents talk to publishers about advertising — discovering inventory, finalising proposals, executing media buys, reporting back. My work here splits into three threads:

  • A seller-side reference agent built on the @adcp/sdk, used as a working baseline for storyboard compliance.
  • Storyboard scenarios and schema fixes for the upstream adcontextprotocol/adcp repository — small contributions that mostly close gaps no one else hit yet.
  • A growing local pack of governance storyboards (token expiry, late approval, self-affiliated advisory) parked until the relevant working groups settle the governance surface.
Active Monorepo

Signals — AdCP signal provider

A small monorepo built around the AdCP signals surface: a shared core, a vanilla provider, a Purrsonality-specific provider, and (soon) a geographic provider seeded from a few million OpenStreetMap points of interest. The goal is a clean reference for how a signal provider answers the get_signals and activate_signal tasks without leaking implementation noise.

Active Indie web

humanMCP

kapoost.humanmcp.net is my personal MCP server — a small endpoint that lets language models (and other agents) ask things about me directly, on my terms, using rotating session codes and a guardian persona that handles operational safety. It is my answer to the question of where personal identity should live in an agentic web.
